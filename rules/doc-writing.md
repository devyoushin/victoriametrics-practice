# 문서 작성 규칙 — victoriametrics-practice

## 기본 원칙
1. 모든 문서는 한국어로 작성하되, 기술 용어(MetricsQL, vmagent, vmalert, vmui 등)는 영어 그대로 사용
2. 제목은 `# [주제] — VictoriaMetrics [문서유형]` 형식으로 통일
3. 문서 상단에 작성일, 작성자, 검토자 메타데이터 포함
4. 코드 블록에는 반드시 언어 지정 (```metricsql, ```yaml, ```bash)
5. Prometheus와의 차이점을 명확히 구분하여 설명

## 문서 구조
- **개요**: 목적과 구성 모드 (3-5문장)
- **구성**: 컴포넌트 설정 및 스토리지 정보
- **운영**: 핵심 명령어 및 체크리스트
- **트러블슈팅**: 증상-원인-해결 테이블
- **참고**: 관련 문서 링크

## MetricsQL 문서화 규칙
- PromQL 호환 함수는 PromQL로 작성
- VM 전용 함수 사용 시 `# VM 전용` 주석 추가
- vmui 접속 URL 및 사용법 포함
- vmagent scrape 설정 예시 포함

## 금지 사항
- 개인 식별 정보 포함 금지
- 스토리지 경로 및 S3 자격증명 문서에 하드코딩 금지
- 검증되지 않은 MetricsQL 쿼리 문서화 금지
